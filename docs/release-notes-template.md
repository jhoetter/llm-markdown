# Release Notes Template

## Summary

- One-paragraph release purpose and scope.

## Added

- New capabilities and APIs.

## Changed

- Behavior changes that are backward compatible.

## Deprecated

- APIs/behaviors scheduled for removal.

## Removed

- Breaking removals.

## Fixed

- Bug fixes with user impact.

## Breaking Change Checklist

- [ ] Document migration path.
- [ ] Update README/docs examples.
- [ ] Add/adjust tests for old + new behavior.
- [ ] Call out compatibility impact in release notes.
- [ ] Verify with `pytest -m "not integration"` and `python -m build`.
